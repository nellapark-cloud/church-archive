# 은혜교회 작업 아카이브 — 사용 설명서

## 이번에 바뀐 점

직접 만든 관리 화면이 GitHub API에 바로 접근하는 방식은 Netlify가 보안상 막고 있어서
작동하지 않았습니다 (계속 401 에러가 났던 이유). 그래서 **Netlify가 공식 지원하는 Decap CMS
방식으로 되돌리고**, 대신 폴더를 **4단계 깊이**까지 중첩해서 만들 수 있게 넉넉하게 구성했습니다.

```
폴더 (1단계) > 폴더 (2단계) > 폴더 (3단계) > 폴더 (4단계) > 자료
```

각 단계에서 "폴더"를 추가할지 "자료"를 추가할지 선택할 수 있어서, 실제로는
"교회 관리 시스템 > 재정 > 2026년 > 상반기 > 자료" 같은 구조도 만들 수 있습니다.

## 구조

```
index.html        방문자가 보는 화면 (폴더를 열면 계속 안으로 들어감)
content/data.json 폴더/자료 구조가 저장되는 파일 (직접 열 필요 없음)
admin/index.html  Decap CMS 로더
admin/config.yml  Decap CMS가 어떤 필드를 보여줄지 정의하는 설정
style.css         공통 디자인
```

## 적용 방법

1. 이 폴더 안의 파일들을 GitHub 저장소에 그대로 덮어써서 업로드해주세요
   ("Add file" → "Upload files" → 이 폴더 내용물 드래그 → "Commit changes")
2. Netlify가 자동으로 재배포합니다 (1~2분)
3. `사이트주소/admin/` 접속 → 로그인

## 관리 화면 사용법

1. 로그인하면 **"작업 아카이브" → "전체 구조 관리"** 클릭
2. **"최상위 항목"** 아래 **"Add"** 버튼을 누르면 **"폴더"**와 **"자료"** 중 선택하는 창이 뜹니다
   - **폴더** 선택 → ID(영문)/이름/설명 입력 → 그 폴더를 펼치면 안에 또 "하위 항목 Add"가 있어서
     그 안에 다시 폴더나 자료를 넣을 수 있습니다 (이렇게 4단계까지 반복 가능)
   - **자료** 선택 → 이름/설명 입력 + 파일 업로드
3. 다 하신 후 오른쪽 위 **"Publish"** 클릭 → 1~2분 뒤 사이트에 반영

## 참고

- ID는 영문과 하이픈만 사용해주세요 (예: `finance-2026`). 화면에 보이는 이름은
  "폴더 이름" 필드에 한글로 편하게 쓰시면 됩니다.
- 4단계보다 더 깊은 구조가 실제로 필요해지면 말씀해주세요, config.yml에 단계를 더 추가해드릴 수 있습니다.
- 예전에 만들었던 "커스텀 관리 화면"(폴더 무제한 중첩 시도)은 이번 버전에서 제거했습니다.
  Netlify의 Git Gateway 정책상 그 방식은 지원되지 않아서, 안정적으로 작동하는 Decap CMS로 통일했습니다.
