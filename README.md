# 은혜교회 작업 아카이브 — 사용 설명서 (최종 안정 버전)

## 지금 이 버전에 대해

여러 방식을 시도했었습니다:
- 커스텀 "구글 드라이브 스타일" 관리 화면 → 버그가 계속 나서 포기
- 진짜 구글 드라이브 연동 → html 파일이 실제로 실행되지 않고 코드/다운로드로만 열려서 포기
  (구글 드라이브는 2015년부터 html을 웹페이지로 렌더링하지 않습니다)

그래서 **확실히 작동하는 걸 확인한 방식**으로 정리했습니다:
- 최상위 폴더는 목록에서 하나씩 클릭해서 들어가는 방식 (깔끔함)
- 그 안의 하위 폴더/자료는 펼치고 접는 방식 (Decap CMS 기본 기능이라 100% 안정적으로 작동)
- 실제 파일은 GitHub에 그대로 저장되어서, 방문자가 열람하면 html이 진짜로 작동합니다

## 구조

```
index.html            방문자가 보는 화면
content/categories/   최상위 폴더마다 파일 하나씩 (관리 화면에서 자동 생성)
content/site.json     사이트 제목/설명
build.js              배포 시 categories + site.json → content/data.json으로 합침
admin/                관리 화면 (Decap CMS)
style.css             공통 디자인
```

## 적용 방법

1. 이 zip의 `church-archive` 폴더 내용물을 GitHub에 통째로 덮어써서 업로드
   - 이전에 있던 `admin/drive-widget.js`는 이제 안 쓰니 GitHub에서 지워주셔도 되고, 안 지우셔도 무해합니다
2. 재배포 확인 (netlify.toml에 빌드 명령이 이미 설정되어 있어서 대시보드 손댈 필요 없음)
3. `/admin/` 재접속 → 로그인

## 관리 화면 사용법

1. 로그인하면 **"최상위 폴더"** 목록이 보입니다 (클릭해야 그 폴더 내용이 열림 — 다른 폴더는 안 보임)
2. 폴더 하나를 열면 **"하위 항목"**이 있고, 거기서 **Add** → **폴더** 또는 **자료** 선택
   - 폴더를 계속 추가하면 그 안에 또 폴더/자료 추가 가능 (총 4단계까지)
   - 자료는 이름/설명 입력 + 파일 업로드(드래그 앤 드롭 가능)
3. **Publish** 클릭 → 1~2분 뒤 사이트에 반영
4. 새 최상위 폴더는 목록 화면에서 **"New 폴더"**
5. 사이트 제목/설명은 왼쪽 메뉴 **"사이트 설정"**

## 공개 사이트 동작

- 일반 방문자에게는 **열람** 버튼만 보입니다
- **다운로드** 버튼은 `/admin/`에 로그인된 상태(같은 브라우저)에서만 보입니다

## 폴더 간 자료 이동

자동 이동 기능은 없습니다. 옮기고 싶으면: 원본 자료 삭제 → 원하는 폴더에서 파일 다시 업로드.
