# 은혜교회 작업 아카이브 — 사용 설명서 (자유 폴더 구조 버전)

## 무엇이 바뀌었나요

이전 버전은 "폴더 > 서랍 > 자료" 딱 3단계로 고정되어 있었습니다.
이번 버전은 **폴더 안에 폴더를 몇 단계든 계속 만들 수 있는 자유 구조**입니다.
그리고 관리 화면을 직접 만들어서, 코드 대신 화면에서:

- 폴더 추가 / 폴더 안에 폴더 추가 (원하는 만큼 깊이)
- 제목·설명 수정 (클릭해서 바로 입력)
- 자료 파일 업로드
- 삭제

를 전부 할 수 있습니다.

## 구조

```
index.html        방문자가 보는 화면 (폴더를 열면 계속 안으로 들어감)
content/data.json 폴더/자료 구조가 저장되는 파일 (이제 직접 열 필요 없음)
admin/index.html  관리 화면 — 여기서 폴더/자료를 추가·수정·삭제
style.css         공통 디자인
```

## 준비 과정 (이미 하셨다면 그대로 유지)

아래 두 가지는 지난번에 이미 설정하셨을 거예요. 그대로 두시면 됩니다.

1. **GitHub 저장소 연결 + Netlify Git 배포** — 이미 완료
2. **Netlify Identity + Git Gateway 활성화, 본인 초대** — 이미 완료

이번 버전은 Decap CMS를 쓰지 않고, 관리 화면이 Git Gateway를 통해 GitHub 저장소에
직접 읽고 씁니다. 그래서 이전에 있던 `admin/config.yml` 파일은 더 이상 필요 없어서 지웠습니다.

## 관리 화면 처음 쓸 때 (딱 한 번만)

1. `사이트주소/admin/` 접속 → 로그인
2. 화면에 "① 저장소 연결 설정" 이라는 상자가 보입니다
3. **GitHub 사용자명**과 **저장소 이름**을 입력합니다
   (예: 지난번 화면에 보였던 `nellapark-cloud` / `church-archive`)
4. 브랜치는 기본값 `main` 그대로 두고, **"저장하고 불러오기"** 클릭

이 정보는 브라우저에 저장되어서, 다음부터는 로그인만 하면 바로 편집 화면이 나옵니다.
(GitHub 저장소 주소는 저장소 페이지 URL — `github.com/사용자명/저장소이름` — 에서 확인 가능합니다)

## 이제부터 자료를 추가/관리하는 방법

- **최상위 폴더 추가**: 맨 아래 "+ 최상위 폴더 추가" 클릭
- **폴더 안에 폴더 추가**: 각 폴더 오른쪽의 "+ 폴더" 클릭 → 그 안에 새 폴더가 생김 (계속 반복 가능, 깊이 제한 없음)
- **자료(파일) 추가**: 폴더 오른쪽의 "+ 자료 업로드" 클릭 → 파일 선택 → 자동 업로드
- **이름/설명 수정**: 제목이나 설명 칸을 그냥 클릭해서 바로 수정
- **삭제**: "삭제" 버튼 (하위 내용도 함께 삭제되니 주의)
- **사이트 제목/설명 수정**: 맨 위 "사이트 제목" / "사이트 설명" 칸에서 수정

**중요**: 화면에서 수정한 내용은 맨 아래 **"변경사항 저장"** 버튼을 눌러야
실제 GitHub에 반영되고, 1~2분 뒤 사이트에 나타납니다. 파일 업로드는 클릭 즉시
저장소에 올라가지만, 폴더 구조·제목·삭제 등은 "변경사항 저장"을 눌러야 확정됩니다.

## 자주 있을 수 있는 상황

- **여러 번 열어서 동시에 수정하면?** 마지막에 저장한 내용이 이전 내용을 덮어씁니다.
  가능하면 한 번에 한 사람만 편집해주세요.
- **실수로 삭제했어요**: GitHub 저장소의 "Commits" 기록에서 이전 버전의 `content/data.json`을
  확인하고 복구할 수 있습니다. (알려주시면 도와드릴게요)
- **폴더 depth가 너무 깊어서 헷갈려요**: 관리 화면에서 들여쓰기로 깊이가 표시되니,
  구조가 복잡해지면 언제든 정리해드릴 수 있어요.
