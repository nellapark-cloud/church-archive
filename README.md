# 은혜교회 작업 아카이브 — 사용 설명서 (구글 드라이브 스타일)

## 이번에 바뀐 점

관리 화면이 예전엔 "모든 폴더가 한 화면에 계속 펼쳐지는" 방식이라 복잡했는데,
이번엔 **최상위 폴더들이 목록으로 뜨고, 그중 하나를 클릭해야 그 폴더 내용이 열리는** 방식으로 바꿨습니다.
구글 드라이브에서 폴더 하나 클릭해서 들어가는 느낌과 비슷합니다.

- 자료 업로드 칸은 원래도 **파일을 끌어다 놓는(드래그 앤 드롭) 방식**을 지원합니다
- 다만 **폴더 간 자료 이동**은 지원하지 않습니다 (삭제 후 원하는 폴더에서 다시 업로드해야 합니다) —
  이 부분은 Netlify가 막고 있는 영역이라 안전하게 구현할 방법이 없었습니다

**참고**: 이번 개편 과정에서 그동안 관리 화면에서 직접 추가하신 "마태복음" 폴더 등 테스트 삼아
만드신 내용은 반영되지 않았습니다 (말씀하신 대로 무시하고 원래 5개 폴더 기준으로 재구성했습니다).

## 구조가 이렇게 바뀌었습니다

```
이전: content/data.json 파일 하나에 모든 폴더/자료가 통째로 들어있었음
이후: content/categories/ 안에 최상위 폴더마다 파일이 하나씩 있음
      (예: content/categories/01-교회-관리-시스템.json)
      + content/site.json (사이트 제목/설명)
      + build.js 가 이 파일들을 합쳐서 content/data.json을 자동으로 만들어줌
      (방문자 화면은 이전과 똑같이 content/data.json을 읽습니다)
```

이렇게 나눠놓으니, 관리 화면에서 "교회 관리 시스템" 폴더를 열면 **그 폴더 내용만** 보이고
다른 4개 폴더는 목록에만 있고 화면에 나타나지 않아서 훨씬 깔끔합니다.

## 적용 방법 (이번엔 한 가지가 추가됩니다)

1. 이 zip의 `church-archive` 폴더 내용물을 GitHub 저장소에 통째로 덮어써서 업로드
   - **주의**: 예전 `content/data.json` 파일은 이번에 지워졌고 `content/categories/` 폴더가 새로 생겼습니다.
     혹시 GitHub에 예전 `content/data.json`이 남아있다면 지워주셔도 되고, 안 지우셔도 무해합니다
2. **이번엔 빌드 스크립트가 추가되어서, 별도 설정 없이 자동으로 실행됩니다**
   (`netlify.toml` 파일에 미리 설정해뒀습니다 — 대시보드에서 손대실 필요 없습니다)
3. Netlify가 재배포하면서 자동으로 `content/data.json`을 새로 만들어줍니다 (1~2분)
4. `/admin/` 재접속 → 로그인

## 관리 화면 사용법

1. 로그인하면 **"최상위 폴더"** 라는 목록이 보입니다 (교회 관리 시스템, 성경공부 자료... 등 5개)
2. 그중 하나를 **클릭** → 그 폴더의 이름/설명과 "하위 항목"만 나옵니다 (다른 폴더는 안 보임)
3. "하위 항목" 아래 **Add** → **폴더** 또는 **자료** 선택
   - 폴더 선택 시 그 안에 또 폴더/자료를 계속 추가 가능 (여러 단계 반복 가능)
   - 자료 선택 시 이름/설명 입력 + **파일을 업로드 칸에 끌어다 놓기(드래그 앤 드롭)**
4. **"Publish"** 클릭 → 1~2분 뒤 사이트에 반영
5. 새 최상위 폴더가 필요하면, 목록 화면에서 오른쪽 위 **"New 폴더"** 클릭
6. 사이트 제목/설명을 바꾸고 싶으면 왼쪽 메뉴의 **"사이트 설정"** 클릭

## 자료를 다른 폴더로 옮기고 싶을 때

지금 방식으로는 자동 이동이 안 되니, 다음과 같이 해주세요:
1. 옮기려는 자료가 있는 폴더를 열어서 파일을 다시 다운로드(또는 갖고 계신 원본 사용)
2. 그 자료 항목을 **삭제**
3. 옮기고 싶은 폴더를 열어서 **같은 파일을 다시 업로드**

번거롭지만, 이 부분까지 자동화하려면 이전에 겪으신 401 에러가 났던 것과 같은 방식(Netlify가 막는 영역)을
다시 시도해야 해서, 안정성을 위해 이 방식으로 유지했습니다.
